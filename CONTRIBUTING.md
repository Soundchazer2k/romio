# Contributing to Romio

Thanks for wanting to help. Romio is community-maintained and there are meaningful ways
to contribute at every skill level.

## The fastest way to contribute (no Rust required)

The BIOS rules database, format compatibility matrix, and emulator status matrix all live
in JSON files under `src-tauri/data/`. If you know that a system's BIOS requirements have
changed, or that a new emulator is now the community recommendation, you can open a PR
that only touches those files.

→ [How to update the BIOS database](docs/contributing/bios-database.md)
→ [How to update the emulator matrix](docs/contributing/emulator-matrix.md)

## Reporting bugs

Use the [bug report template](.github/ISSUE_TEMPLATE/bug_report.md).
Include your OS, the frontend you're targeting, and what Romio reported vs. what you expected.
